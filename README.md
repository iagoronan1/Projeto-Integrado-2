# Projeto-Integrado-2
Conexão Escola e Família — README
Visão Geral
Este projeto é um Sistema de Gestão Escolar desenvolvido em Python com foco na Programação Orientada a Objetos (POO). Ele foi criado com fins acadêmicos para aplicar, de forma prática, os conceitos estudados na disciplina.
O sistema representa um ambiente escolar simples, permitindo organizar informações sobre alunos, professores, responsáveis, turmas e disciplinas.
________________________________________
Objetivo do Projeto
•	Aplicar os principais conceitos de POO na prática;
•	Modelar um problema real (gestão escolar) usando classes;
•	Desenvolver um código organizado, claro e funcional;
•	Atender aos critérios avaliativos da disciplina.
________________________________________
Estrutura do Sistema
O projeto é composto por classes que representam entidades do mundo real, como:
•	Administração Escolar
•	Aluno
•	Professor
•	Responsável
•	Turma
•	Disciplina
•	Tarefa
•	Comunicação
Cada classe possui uma responsabilidade bem definida dentro do sistema, o que facilita o entendimento e a manutenção do código.
________________________________________
Princípios de POO Utilizados
O projeto foi desenvolvido com base nos principais princípios da Programação Orientada a Objetos:
•	Abstração: representação das entidades do mundo real apenas com informações relevantes;
•	Encapsulamento: os dados são manipulados por meio de métodos, evitando acessos indevidos;
•	Responsabilidade única: cada classe possui uma função específica;
•	Baixo acoplamento: as classes se comunicam de forma controlada;
•	Alta coesão: os métodos e atributos de cada classe estão relacionados ao seu objetivo.
________________________________________
Funcionamento Geral
A administração escolar é responsável pelos cadastros. Professores registram notas e frequência, enquanto alunos e responsáveis podem consultar essas informações. O sistema funciona totalmente em memória, sem uso de banco de dados ou interface gráfica.
________________________________________
Testes Realizados
O sistema foi testado por meio da execução do fluxo principal, verificando:
•	Criação correta dos objetos;
•	Funcionamento dos métodos principais;
•	Interação entre as classes.
________________________________________
Possíveis Usos da Nossa Solução
Mesmo sendo um projeto acadêmico, o sistema pode servir como base para escolas pequenas, cursos ou projetos educacionais que precisem de um controle simples de alunos, turmas e desempenho escolar.
________________________________________
Organização da Equipe
O desenvolvimento foi feito de forma colaborativa, com divisão de tarefas como levantamento de informações, modelagem do sistema, implementação do código, testes e documentação.
